ScottyLabs Dining API

Delicious, delicious CMU data.

Documentation

The Dining API is an endpoint that allows you to get information about a given eatery on Carnegie Mellon's campus. In particular, we offer location, time information, keywords, and a short description you can use in your CMU Dining related applications.

While more detailed documentation follows, go ahead and try it out by going to apis.scottylabs.org/dining/v1/locations in your favourite browser.

Version 1

In version 1, there is only one endpoint, at /locations.

GET /locations

Request:

GET apis.scottylabs.org/dining/v1/locations

Response format:

{
  "locations": [{
    "name": string,
    "description": string,
    "keywords": [string],
    "location": string,
    "times": [{
      "start": {
       "day": number,
       "hour": number,
       "minute": number,
      },
      "end": {
        "day": number,
        "hour": number,
        "minute": number,
      }
    }, ...]
  }, ...]
}
Note: The day fields conform to JavaScript's Date.getDay() format, where day 0 is Sunday, while day 6 is Saturday.
